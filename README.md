## Мета роботи
Розробити повноцінний клієнт-серверний застосунок на мові Java, що реалізує бізнес-логіку управління аграрними ресурсами (AgroTech). Проєкт поєднує роботу з базою даних, REST API та інтеграцію з зовнішніми сервісами.

## Технологічний стек
* **Backend:** Java 17, Spring Boot 3.x
* **Database:** H2 Database (In-memory)
* **ORM:** Spring Data JPA (Hibernate)
* **Frontend:** Thymeleaf (Server-side rendering)
* **API Parsing:** RestTemplate (JSON processing)

## Архітектура
### Шари застосунку
* **Controller:** Обробка HTTP-запитів та маршрутизація.
* **Service:** Бізнес-логіка та взаємодія з зовнішнім API.
* **Repository:** Взаємодія з базою даних через JPA.
* **Model:** Опис сутностей (Entities).

## База даних
Використовується реляційна модель зі зв'язком **One-to-Many**:
* **Таблиця `farms`:** назва, широта, довгота.
* **Таблиця `sensors`:** тип датчика, статус активності, id ферми.

## Функціонал
* ✅ **CRUD операції:** Повне управління даними про ферми та датчики.
* 🔍 **Моніторинг:** Перегляд стану об'єктів у реальному часі.
* 🌐 **Парсинг зовнішнього API:** Автоматичне отримання погоди через Open-Meteo.
* 🎭 **DTO патерн:** Використання Records для безпечної передачі даних.

## Структура проєкту
Проєкт організований за пакетним принципом:
- `controller/` — обробка запитів.
- `service/` — логіка та парсинг.
- `repository/` — доступ до БД.
- `model/` — сутності бази даних.

## Результати
Застосунок успішно інтегрує дані з БД та зовнішнього API, відображаючи актуальну температуру для обраних локацій (Ulsteinvik, Kyiv) на фронтенд-панелі.
