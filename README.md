# AgroManager — Платформа управління сільськогосподарськими ресурсами

1. Мета роботи
Розробка повноцінного клієнт-серверного застосунку на базі Spring Boot, що реалізує логіку управління агро-підприємствами. Метою є інтеграція роботи з реляційною базою даних, створення REST API та впровадження модуля парсингу даних із зовнішніх джерел для автоматизації моніторингу погодних умов.

2. Технологічний стек
Мова програмування: Java 17

Фреймворк: Spring Boot 3.x

Доступ до даних: Spring Data JPA (Hibernate)

База даних: H2 Database (In-memory)

Шаблонізатор (UI): Thymeleaf

Зовнішні API: Open-Meteo (REST JSON Parsing)

Інструменти збірки: Maven

3. Архітектура
Застосунок побудований за багатошаровою архітектурою (Layered Architecture):

Controller Layer: Обробка HTTP-запитів та повернення JSON (REST) або HTML-сторінок.

Service Layer: Містить бізнес-логіку, зокрема взаємодію із зовнішнім погодним сервісом.

Repository Layer: Рівень доступу до бази даних через інтерфейси JPA.

DTO Layer: Використання Java Records для безпечної передачі даних між сервісом та контролером.

Model (Entity) Layer: Опис структури таблиць бази даних.

4. База даних
У проекті реалізовано реляційну модель даних із використанням зв’язку One-to-Many (Один до багатьох):

Farm (Ферма): Зберігає назву та географічні координати (latitude, longitude).

Sensor (Датчик): Зберігає тип датчика та його статус. Кожен датчик прив'язаний до конкретної ферми через зовнішній ключ (farm_id).

5. Функціонал
RESTful API: Повний доступ до даних про ферми через endpoint /api/farms.

External Data Parsing: Автоматичне отримання актуальної температури повітря в реальному часі для кожної локації (наприклад, Ulsteinvik та Kyiv) за допомогою RestTemplate.

Web Dashboard: Візуалізація списку об'єктів у браузері з відображенням "спарсених" даних.

Auto-initialization: Автоматичне наповнення бази даних тестовими записами при старті програми.

7. Результати
В результаті розробки створено стабільний Java-застосунок, який:

Успішно підключається до вбудованої БД H2.

Надає дані у форматі JSON для сторонніх клієнтів.

Динамічно обробляє дані з інтернету та виводить їх на фронтенд частину.

Коректно відображає локальні дані (наприклад, погоду в м. Ульстейнвік).

8. Висновки
Під час виконання лабораторної роботи було опановано принципи побудови Spring Boot застосунків. Реалізовано інтеграцію декількох шарів архітектури, налагоджено роботу з ORM (Hibernate) та впроваджено механізм споживання зовнішніх API. Застосунок є масштабованим і може бути використаний як основа для дипломного проекту в галузі AgroTech.
